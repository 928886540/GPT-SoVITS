# Evaluation Plan

## 目标

验证 GPT-SoVITS 官方实现与 Genie-TTS 是否适合作为新的专业本地 TTS 产品底座。

不要求兼容旧 IndexTTS2 接口。评估以交付质量和运行稳定性为准。

交付边界：

- 不评估公网服务能力。
- 不以远程 API 托管为目标。
- 只评估用户本机可运行、可打包、可复现的能力。
- 本机 HTTP API 可以存在，但只能算内部实现细节。

## 候选路线

### A. 官方 GPT-SoVITS

验证项：

- 官方安装路径是否能稳定复现。
- `api_v2.py` 是否能直接作为服务入口。
- `streaming_mode` 的不同档位首包延迟、音质和资源占用。
- 角色级参考音频切换成本。
- 独立 GPT/SoVITS 权重切换成本。
- 中文、日语、中日混合质量。

### B. Genie-TTS

验证项：

- ONNX 推理是否显著降低显存/内存压力。
- 本机 FastAPI 或脚本入口是否容易启动、停止和打包。
- `load_character` / `set_reference_audio` / `tts` 是否适合多角色音色。
- `/tts` 返回的流是否是真生成流。
- 多角色预加载后的切换延迟。
- 中文、日语、中日混合质量。

## 通过标准

硬性标准：

- 12GB 显存机器不应频繁 OOM。
- 不应依赖 C 盘默认缓存大量写入。
- 单角色短句首包延迟可接受。
- 多角色连续对话不应出现明显长时间空白。
- 本机入口稳定，必要时可通过 `127.0.0.1` API 调用。

优先标准：

- 支持中文和日语。
- 支持角色级音色配置。
- 支持流式播放或可用的准流式播放。
- 支持清晰的模型/参考音频管理。
- 便于后续包装成本地整合包。

## 测试矩阵

文本类型：

- 中文短句：20-40 字。
- 中文长段：300-800 字。
- 日语短句：1-2 句。
- 中日混合：中文旁白 + 日语台词。
- 多角色对话：旁白 + 用户 + 两个角色。

指标：

- 冷启动耗时。
- 首包音频耗时。
- 完整生成耗时。
- RTF。
- 显存峰值。
- 系统内存峰值。
- 磁盘缓存增长。
- 失败率。
- 主观音质。
- 音色相似度。
- 角色切换稳定性。
