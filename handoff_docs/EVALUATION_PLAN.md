# Evaluation Plan

## 目标

验证 GPT-SoVITS 官方实现与 Genie-TTS 是否适合作为新的专业本地 TTS 产品底座。

不要求兼容旧 IndexTTS2 接口。评估以交付质量和运行稳定性为准。

交付边界：

- 不评估公网服务能力。
- 不以远程 API 托管为目标。
- 只评估用户本机可运行、可打包、可复现的能力。
- 本机 HTTP API 可以存在，但只能算内部实现细节。

## 候选路线

### A. 官方 GPT-SoVITS

验证项：

- 官方安装路径是否能稳定复现。
- `api_v2.py` 是否能直接作为服务入口。
- `streaming_mode` 的不同档位首包延迟、音质和资源占用。
- 角色级参考音频切换成本。
- 独立 GPT/SoVITS 权重切换成本。
- 中文、日语、中日混合质量。

### A1. GPT-SoVITS 三档模式

#### 1. Zero-shot 参考音频模式

当前已完成的工作属于这一档。

- 输入：5-10 秒参考音频 + `prompt_text`。
- 不训练权重，直接使用官方预训练模型。
- 用途：快速筛音色、验证 Tavo adapter、流式首包、缓存和多角色接口。
- 当前证据：
  - `reports/OFFICIAL_V2_FIRST_BENCH.md`
  - `reports/TAVO_ADAPTER_OFFICIAL_BINDING.md`
  - `reports/REAL_VOICE_GPTSOVITS_FIRST.md`

通过标准：

- 能稳定生成 WAV。
- 首包低于完整生成时间，当前卡片可先播。
- 相同输入能命中 cache。
- Voice Profile 能保存 `ref_audio_path`、`prompt_text`、参数和权重路径。

#### 2. Few-shot 少量微调模式

下一阶段要验证这一档。

- 输入：同一说话人 1-5 分钟素材。
- 官方 README 说明约 1 分钟数据即可 few-shot 微调，提升相似度和真实感。
- 流程：素材收集 -> 切片 -> 降噪/去混响可选 -> ASR -> 人工校对 -> GPT/SoVITS 微调 -> 推理对比。
- 目标：验证 12GB 显存机器能否稳定微调，训练后是否明显优于 zero-shot。

最小实验：

- 候选音色：从 `prompts/library/女声/` 或用户提供 ASMR 素材中选 1 个。
- 素材长度：先做 1-3 分钟，不直接上 40 分钟。
- 模型版本：先 v2 或 v2ProPlus，不直接押 v4。
- 记录：训练耗时、显存峰值、loss/日志、输出权重路径、推理首包、RTF、人工听感。

通过标准：

- 训练不 OOM。
- 生成稳定性不低于 zero-shot。
- 音色相似度或 ASMR 细节有明显提升。
- 权重切换成本可接受，能纳入 Tavo Voice Profile。

#### 3. Full / Character Model 完整角色模型模式

最后阶段才做这一档。

- 输入：同一说话人 20-40 分钟以上高质量素材。
- 目标：沉淀可分发的角色级 GPT/SoVITS 权重。
- 用途：核心 ASMR 角色、长期复用角色、多段长文本稳定生成。
- 成本：素材清洗、逐字稿校对、训练、质检和包体管理最高。

进入条件：

- few-shot 已证明该音色方向值得继续。
- 有足够干净、单人、低噪、低混响素材。
- 已有可复现训练配置和评估文本。

通过标准：

- 长文本稳定，少漏字/重复。
- 音色一致性优于 zero-shot/few-shot。
- ASMR 场景下呼吸、贴耳、低语不会明显破音或金属音。
- 权重、配置、参考音频和 Profile 能打包给本地用户。

### A2. 版本选择计划

- v2：当前基准线，已跑通官方 API、adapter、流式和真实音色 zero-shot。
- v2Pro / v2ProPlus：下一优先下载和对比。官方说明其硬件成本和速度接近 v2，效果可超过 v4，适合 12GB 显存本机路线。
- v3 / v4：更偏参考音频本身，对训练集质量更敏感。官方说明平均质量数据集上 v1/v2/v2Pro 可能更稳，v3/v4 不一定适合低质量数据。只在同素材对比后决定是否进入 ASMR 主线。

### B. Genie-TTS

验证项：

- ONNX 推理是否显著降低显存/内存压力。
- 本机 FastAPI 或脚本入口是否容易启动、停止和打包。
- `load_character` / `set_reference_audio` / `tts` 是否适合多角色音色。
- `/tts` 返回的流是否是真生成流。
- 多角色预加载后的切换延迟。
- 中文、日语、中日混合质量。

## 通过标准

硬性标准：

- 12GB 显存机器不应频繁 OOM。
- 不应依赖 C 盘默认缓存大量写入。
- 单角色短句首包延迟可接受。
- 多角色连续对话不应出现明显长时间空白。
- 本机入口稳定，必要时可通过 `127.0.0.1` API 调用。

优先标准：

- 支持中文和日语。
- 支持角色级音色配置。
- 支持流式播放或可用的准流式播放。
- 支持清晰的模型/参考音频管理。
- 便于后续包装成本地整合包。

## 测试矩阵

文本类型：

- 中文短句：20-40 字。
- 中文长段：300-800 字。
- 日语短句：1-2 句。
- 中日混合：中文旁白 + 日语台词。
- 多角色对话：旁白 + 用户 + 两个角色。

指标：

- 冷启动耗时。
- 首包音频耗时。
- 完整生成耗时。
- RTF。
- 显存峰值。
- 系统内存峰值。
- 磁盘缓存增长。
- 失败率。
- 主观音质。
- 音色相似度。
- 角色切换稳定性。

训练额外指标：

- 素材总时长。
- 有效切片数量。
- ASR 字错率或人工校对状态。
- 训练耗时。
- 训练显存峰值。
- 输出 GPT 权重路径。
- 输出 SoVITS 权重路径。
- 权重包体积。
- 与 zero-shot 同文本对比听感。
